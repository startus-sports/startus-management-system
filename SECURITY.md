## 3. 各対策の詳細

### 3.1 データベースアクセス制御（RLS: Row Level Security）

**問題:** 以前は、ログインしたユーザーであれば誰でも全てのデータにアクセス・変更が可能でした。

**対策:** データベースの各テーブルに「行レベルセキュリティ（RLS）」を設定し、在籍スタッフのみがデータにアクセスできるようにしました。

**対象テーブル（10テーブル）:**
- 会員情報（members）
- 申請情報（applications）
- スタッフ情報（staff）
- 教室情報（classrooms）
- 活動ログ（activity_log）
- アプリ設定（app_config）
- チャット関連（chat_channels, chat_channel_members, chat_messages）
- 申請コメント（application_comments）

**効果:**
- ログインしていない人 → データ閲覧不可
- 在籍スタッフ以外のGoogleアカウントでログイン → データ閲覧不可
- 退職したスタッフ → ステータスが「在籍」でなくなった時点でアクセス不可

---

### 3.2 管理者専用操作の保護

**問題:** 以前は、管理者かどうかのチェックがブラウザ上のみで行われており、技術的に回避可能でした。

**対策:** データベース側で管理者チェックを行うようにしました。管理者でないスタッフが管理者専用操作を試みても、データベースが拒否します。

| 操作 | 一般スタッフ | 管理者 |
|------|:----------:|:-----:|
| 会員情報の閲覧・編集 | OK | OK |
| 申請の閲覧・処理 | OK | OK |
| チャットの利用 | OK | OK |
| 会員の削除 | NG | OK |
| スタッフの追加・編集・削除 | NG | OK |
| 教室の追加・編集・削除 | NG | OK |
| システム設定の変更 | NG | OK |

---

### 3.3 セキュリティヘッダー

**対策:** Webサーバーに以下のセキュリティヘッダーを追加しました。

| ヘッダー | 効果 |
|----------|------|
| Content-Security-Policy | 許可されたサイト以外からのスクリプト読み込みを防止 |
| X-Frame-Options: DENY | 他のサイトにこのアプリを埋め込まれることを防止 |
| X-Content-Type-Options | ファイルの種類を偽装する攻撃を防止 |
| Referrer-Policy | 外部サイトへのアクセス時にURLが漏れることを防止 |
| Permissions-Policy | カメラ・マイク・位置情報の不正利用を防止 |

---

### 3.4 外部ライブラリの整合性チェック（SRI）

**問題:** Excelエクスポートに使用している外部ライブラリ（SheetJS）が、提供元で改ざんされた場合にそのまま読み込んでしまうリスクがありました。

**対策:** SRI（Subresource Integrity）ハッシュを設定し、ファイルが改ざんされた場合はブラウザが自動的に読み込みを拒否するようにしました。

---

### 3.5 XSS対策

**問題:** 通知メッセージ（トースト）の表示処理で、理論上は悪意あるスクリプトが実行される可能性がありました。

**対策:** HTML直接挿入（innerHTML）からテキスト安全挿入（textContent）に変更し、スクリプトが実行されない安全な方法に修正しました。

---

### 3.6 個人情報のログ出力停止

**問題:** ブラウザの開発者ツール（F12）のコンソールに、ログインユーザーのメールアドレスとスタッフ情報が表示されていました。

**対策:** 該当のログ出力を削除しました。

---

### 3.7 会員写真の非公開化

**問題:** 会員写真の保存先（Supabase Storage）が公開設定になっており、URLを知っている人であれば誰でもアクセスできる状態でした。

**対策:** ストレージバケットを非公開（Private）に変更しました。ログイン済みのスタッフのみが写真を閲覧できます。

---

### 3.8 APIキーの利用制限

**問題:** Googleカレンダー連携に使用しているAPIキーに利用制限がなく、キーを知っている人がどこからでも使用できる状態でした。

**対策:** HTTPリファラー制限を設定し、以下のサイトからのみ利用可能にしました:
- `https://member-manager-nu.vercel.app`（本番環境）
- `http://localhost`（開発環境）

## 4. 職員向け注意事項

### やってはいけないこと

- **パスワードやログイン情報を他人と共有しない** — 各自のGoogleアカウントでログインしてください
- **公共のPCでログインしたままにしない** — 使用後は必ずログアウトしてください
- **不審なメールのリンクからログインしない** — 必ずブックマークまたは直接URLを入力してアクセスしてください
- **会員情報のスクリーンショットをSNSに投稿しない** — 個人情報保護法に抵触します

### 管理者向け

- スタッフが退職した場合は、速やかにスタッフテーブルのステータスを「退職」に変更してください（これだけでシステムへのアクセスが遮断されます）
- 新しいスタッフを追加する際は、管理者権限（is_admin）の付与を慎重に判断してください
- 定期的にスタッフ一覧を確認し、不要なアカウントがないか確認してください

# STARTUS スポーツアカデミー システム統合計画（職員向け）

## この資料について

現在、クラブの業務システムは複数のツール（SharePoint、Google Calendar、Sgrum、Google フォーム等）に分かれています。これらを**1つの管理画面（STARTUS Management System）に統合**し、業務を効率化する計画です。

この資料では、**何が変わるのか・何が変わらないのか・いつ変わるのか**を説明します。

---

## 現在のシステム構成（いまの状態）

```
┌───────────────────────────────────────────────────┐
│  日常業務で使っているツール                          │
├───────────────────────────────────────────────────┤
│                                                   │
│  📋 SharePoint ClassList（教室マスタ）              │
│     → 教室の一覧・定員・曜日・会場などを管理         │
│     → フォームの教室選択肢もここから取得              │
│                                                   │
│  📅 Google Calendar + Sgrum（スケジュール）        │
│     → Sgrum でスケジュール入力                    │
│     → Google Calendar に反映                       │
│     → HP の各教室ページにカレンダー表示              │
│     → 体験フォーム等で日程表示                      │
│                                                   │
│  📝 Google フォーム + GAS（各種申請）                │
│     → 体験申込・入会・退会・振替・変更フォーム         │
│     → SharePoint Lists からタグ情報を取得            │
│                                                   │
│  💻 会員管理アプリ（member-manager）                 │
│     → 会員一覧・申請管理・体験管理・月謝管理          │
│                                                   │
│  💻 スケジュール管理アプリ（calendar-manager）        │
│     → スケジュールの一括生成・管理（開発中）           │
│                                                   │
└───────────────────────────────────────────────────┘
```

### 現在の課題

| 課題 | 内容 |
|------|------|
| ツールが多すぎる | SharePoint、Google Calendar、Sgrum、Forms など複数を行き来 |
| 教室情報が分散 | SharePoint・会員管理アプリ・スケジュールアプリの3箇所に同じ情報 |
| スケジュール入力が二重 | Sgrum と スケジュール管理アプリで同じ作業が必要になる可能性 |
| データの食い違い | 複数の場所にある情報がずれるリスク |
| アカウントが個人依存 | 松井個人のアカウントで運用しており、組織として管理できていない |

## 統合後のシステム（目指す姿）

```
┌─────────────────────────────────────────────────────────┐
│  STARTUS 統合管理アプリ（会員管理アプリを拡張）            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  👥 会員管理     → 会員一覧・検索・編集（今と同じ）        │
│  📅 スケジュール  → スケジュール入力・管理（★新機能）      │
│  ✅ 出席管理     → 出席記録・確認（★新機能）             │
│  📋 申請管理     → 体験・入会・退会等の申請処理（今と同じ） │
│  🏫 教室マスタ   → 教室情報の一元管理（★統合）           │
│  💰 月謝管理     → 月謝の管理（今と同じ）                │
│                                                         │
└────────────────────┬────────────────────────────────────┘
                     │
                     │ 自動連携
                     ▼
          ┌──────────────────────┐
          │ Google Calendar      │
          │ → HP表示用に自動同期  │
          │ → 見る専用（入力不要） │
          └──────────────────────┘
```

### 統合後のメリット

| メリット | 内容 |
|----------|------|
| 管理画面が1つに | 会員もスケジュールも出席も1つのアプリで完結 |
| 教室情報は1箇所 | 「マスタ」タブで変更すれば全体に反映 |
| スケジュール入力が楽に | 年間一括生成、曜日指定で自動作成 |
| Google Calendar は自動 | アプリで「公開」にすれば自動でカレンダーに反映 |
| Sgrum 不要に | スケジュールアプリが代替 |
| 組織アカウントで管理 | クラブのアカウントで安全に運用 |

## 職員の皆さんへの影響

### 変わること

| 時期 | 変更内容 | 影響 |
|------|----------|------|
| Phase 1 | スケジュール入力を Sgrum → 管理アプリに変更 | **入力方法が変わる**（研修あり） |
| Phase 2 | 教室情報の変更を SharePoint → 管理アプリに変更 | **教室追加・変更の手順が変わる** |
| Phase 5 | 出席管理機能が追加 | **新機能として使い始める** |

### 変わらないこと

| 項目 | 説明 |
|------|------|
| HP のカレンダー表示 | 引き続き Google Calendar ベースで表示（裏側が変わるだけ） |
| 保護者向けフォーム | 体験申込・入会フォームの見た目・URLは変わらない |
| 会員管理の操作 | 会員一覧・検索・編集は今と同じ |
| 申請処理の操作 | 申請の承認・却下は今と同じ |

## よくある質問

**Q: 保護者に影響はありますか？**
A: ありません。フォームのURL・見た目・操作方法は変わりません。

**Q: Google Calendar は使えなくなりますか？**
A: 表示用としてそのまま使えます。入力方法が Sgrum → 管理アプリに変わるだけです。

**Q: スマホから使えますか？**
A: はい。管理アプリはスマホのブラウザからもアクセスできます。

**Q: 途中で問題が起きたら元に戻せますか？**
A: はい。各フェーズで Google Calendar 同期は維持されるため、いつでも以前の方法に戻せます。

## 1. Git / GitHub の仕組み

### Git とは？

**ファイルの変更履歴を記録するツール**です。

```
例えるなら...

  普通のファイル保存:
    report.docx  →  上書き保存  →  前の内容は消える

  Git:
    report.docx  →  変更を記録  →  いつでも過去の状態に戻せる
                     (v1)  (v2)  (v3)  ← 全部残っている
```

### GitHub とは？

**Gitの記録をインターネット上に保管するサービス**です。

```
┌─────────────────┐          ┌─────────────────┐
│   あなたのPC     │  push →  │    GitHub        │
│  （ローカル）     │          │ （クラウド上）    │
│                 │  ← pull  │                 │
│  コードを編集    │          │  バックアップ     │
│  Gitで履歴管理   │          │  どこからでも閲覧 │
└─────────────────┘          └─────────────────┘
```

**メリット**:
- PCが壊れてもコードが失われない
- 変更履歴をオンラインで確認できる
- スマホからもコードを確認できる

### 基本用語（3つだけ覚えればOK）

| 用語 | 意味 | 例え |
|------|------|------|
| **commit**（コミット） | 変更を記録する | 「セーブする」 |
| **push**（プッシュ） | ローカルの記録をGitHubに送る | 「クラウドにアップロード」 |
| **pull**（プル） | GitHubの記録をローカルに取得する | 「クラウドからダウンロード」 |

```
作業の流れ:

  ファイルを編集
      ↓
  commit（変更を記録）   ← 「この変更をセーブ」
      ↓
  push（GitHubに送信）   ← 「クラウドにアップロード」
```

> **このプロジェクトでは commit と push は自動化されているので、手動で行う必要はありません。**

## 3. 自動化フロー

### 全体の流れ（すべて自動）

```
あなたが Claude Code で変更を依頼
    ↓
Claude がコードを編集（Edit / Write）
    ↓  ← ここから全自動
[自動] git commit（変更を記録）
    ↓
[自動] git push（GitHubに送信）
    ↓
[自動] Vercel がデプロイ（サイトを更新）
    ↓
https://member-manager-nu.vercel.app が更新される
```

**つまり**: Claude に「〇〇を変更して」と言うだけで、コード変更 → GitHub保存 → サイト更新 まで全自動です。

### スマホから編集する3つの方法

#### 方法1: スマホの Claude Code アプリ → GitHub接続（おすすめ）

Claude Code アプリから GitHub リポジトリに直接接続して編集できます。
PCと同じように Claude にお願いするだけでコード編集が可能です。

```
スマホの Claude Code アプリ → GitHub リポジトリに接続
    ↓
Claude にコード変更を依頼 → GitHub に直接コミット
    ↓
Vercel が自動デプロイ → サイト更新
    ↓
次に PC で Claude Code を開く
    ↓
[自動] git pull（最新コードを取得） → そのまま作業OK
```

#### 方法2: GitHub のWeb画面/アプリで手動編集

GitHubのWeb画面やアプリからコードを直接編集することもできます。

```
スマホで GitHub 上のファイルを編集
    ↓
GitHub に保存 → Vercel が自動デプロイ（サイト更新）
    ↓
次に PC で Claude Code を開く
    ↓
[自動] git pull（最新コードを取得） → そのまま作業OK
```

**手順**: GitHub → ファイルを開く → 鉛筆アイコン → 編集 → 「Commit changes」

#### 方法3: PC に接続（リモート）

スマホの Claude Code から PC にリモート接続すれば、PC上のプロジェクトを直接編集できます。
PC側の hooks が動くので、自動 commit & push も機能します。

#### 3つの方法の比較

| 方法 | Claudeに依頼 | 自動デプロイ | PC同期 |
|---|---|---|---|
| **Claude Code → GitHub接続** | できる | 自動 | 自動 |
| **GitHub で手動編集** | 自分で書く | 自動 | 自動 |
| **Claude Code → PC接続** | できる | 自動 | 不要（直接編集） |

> **注意**: スマホとPCで**同時に同じファイル**を編集するとコンフリクト（衝突）が起きます。
> スマホでの編集が終わってからPCで作業を始めましょう。

## 4. 日常の運用方法

### 普段の作業（Claude Code使用時）

**やること**: Claude Code にやりたいことを伝えるだけ。

```
あなた: 「会員一覧の表示を修正して」
Claude: （コードを編集）
        → 自動で commit & push
        → サイトが更新される
```

Git の知識は不要です。

### 手動でコミットしたい場合

Claude Code で `/commit` と入力するだけです。
Claude が変更内容を分析して適切なコミットメッセージを自動生成します。

### 変更履歴を確認する方法

**方法1: GitHubのWeb画面（おすすめ）**
1. https://github.com/mattun9/member-manager にアクセス
2. 「Commits」または時計アイコンをクリック
3. いつ、どんな変更があったか一覧で確認できる

**方法2: コマンドで確認**
```bash
git log --oneline -10    # 直近10件の変更履歴を表示
```

### スマホから確認する方法

**アプリを使う（GitHub Mobile）**:
1. App Store / Google Play から「GitHub」アプリをインストール
2. mattun9 アカウントでログイン
3. member-manager リポジトリを開く
4. コードやコミット履歴を確認できる

**ブラウザから**:
- https://github.com/mattun9/member-manager にアクセス

### デプロイ状況の確認

Vercel が正常にデプロイできたかは以下で確認できます：

```bash
npx vercel ls    # 最新のデプロイ一覧を表示
```

または GitHub のリポジトリページで、最新のコミットの横に緑のチェックマーク（成功）か赤のバツ（失敗）が表示されます。

## 5. よくあるケースと対処法

### ケース1: 変更を元に戻したい

**直前の変更を取り消したい場合:**
```
Claude Code に「直前の変更を元に戻して」と伝える
```
Claude が `git revert` を使って安全に元に戻します。

**特定の時点まで戻したい場合:**
```
Claude Code に「○○の変更前の状態に戻して」と伝える
```
Claude が履歴を確認して対応します。

> 自分で `git reset --hard` は使わないでください（データが消える可能性があります）。

### ケース2: 間違えてコミットしてしまった

**問題ないです。** Private リポジトリなので自分しか見えません。
修正したい場合は Claude Code に「さっきの変更を修正して」と言えばOKです。

### ケース3: 自動コミットが動かない

考えられる原因と対処:

| 原因 | 対処 |
|------|------|
| インターネット接続なし | Wi-Fiを確認。接続復帰後に `git push` で送信される |
| GitHubの認証が切れた | `gh auth login` で再ログイン |
| フックの設定が消えた | `.claude/settings.local.json` の hooks 設定を確認 |

### ケース4: 複数のPCで作業する場合

別のPCで作業する場合は、最初に GitHub からコードを取得します：

```bash
# 初回のみ（別のPCにコードをコピー）
git clone https://github.com/mattun9/member-manager.git

# 2回目以降（最新のコードを取得）
cd member-manager
git pull
```

**注意**: 2台のPCで同時に同じファイルを編集すると「コンフリクト（衝突）」が起きることがあります。基本的に1台のPCで作業するようにしましょう。

### ケース5: インターネット接続がない場合

- **ファイルの編集**: 問題なくできます（ローカルで動作するため）
- **commit**: ローカルに保存されます（問題なし）
- **push**: インターネット復帰後に `git push` を実行すれば送信されます
- **Vercelデプロイ**: push完了後に自動で行われます

### ケース6: ファイルを誤って削除してしまった

**Git で追跡されているファイルなら復元できます:**

```
Claude Code に「○○.js を直前のコミットの状態に復元して」と伝える
```

または:
```bash
git checkout -- ファイル名    # 直前のコミットの状態に復元
```

### ケース7: Vercelのデプロイが失敗した

1. GitHub のリポジトリページでコミット横のステータスを確認
2. 赤いバツが出ている場合、クリックして詳細ログを確認
3. Claude Code に「Vercelのデプロイが失敗しているので原因を調べて」と伝える

よくある原因:
- JavaScriptの構文エラー（書き方のミス）
- 参照しているファイルが見つからない

### ケース8: GitHubの認証が切れた

```bash
# 認証状態を確認
gh auth status

# 再ログイン（ブラウザが開く）
gh auth login
```

### ケース9: 大量のファイルを一度に変更した

自動コミットは変更のたびに実行されますが、大量の変更をまとめてコミットしたい場合は：

```bash
git add -A
git commit -m "まとめてコミット: ○○の変更"
git push
```

### ケース10: 自動コミット＆プッシュを設定したい（新しいPCやスマホ）

新しい環境で Claude Code を使い始める時に、自動同期を設定します：

```bash
bash scripts/setup-auto-sync.sh
```

これで Claude Code がコード変更するたびに自動で commit → push → Vercel デプロイされます。

### ケース11: Supabase にSQLを実行したい

マイグレーションファイルやデータ追加SQLを実行する場合：

```bash
bash scripts/run-sql.sh migration-xxx.sql
```

初回は Supabase CLI のセットアップが必要です。スクリプトがガイドします。

## 6. 注意点・やってはいけないこと

### 絶対にやってはいけないこと

| やってはいけないこと | 理由 | 危険度 |
|---|---|---|
| リポジトリを **Public に変更** する | `config.js` にAPIキー（Supabase, Google）が含まれているため、公開すると悪用される | **致命的** |
| `git push --force` を実行する | 他のコミット履歴が消える可能性がある | **高** |
| `git reset --hard` を実行する | ローカルの変更が全て消える | **高** |
| `.gitignore` のファイルを手動で add する | `.claude/` や `.vercel/` にはローカル固有の設定があり、共有すべきでない | **中** |

### 注意すべきこと

| 項目 | 説明 |
|---|---|
| **config.js の扱い** | APIキーが含まれる。Privateリポジトリなので現状は問題ないが、絶対にPublicにしないこと |
| **大きなファイル（画像・動画）** | GitHubは100MB以上のファイルをアップロードできない。画像は外部サービス（Cloudinary等）を使う |
| **同時編集** | 2つのClaude Codeセッションで同時に同じファイルを編集すると衝突する可能性がある |
| **SQLファイル** | `migration-*.sql` や `setup-*.sql` は一度実行したら二度実行しないこと（データが重複する） |

### もし Public にしたくなった場合

将来的にコードを公開したい場合は、事前に以下の対応が必要です：

1. `config.js` のAPIキーを環境変数に移す
2. `.gitignore` に `config.js` を追加
3. `config.example.js` をテンプレートとして用意
4. Git履歴からも機密情報を削除する（`git filter-branch` 等）
5. Vercelの環境変数にAPIキーを設定

> これらは Claude Code に依頼すれば対応できます。

## 7. 便利なコマンド一覧（チートシート）

### 基本操作

| やりたいこと | コマンド |
|---|---|
| 現在の状態を確認 | `git status` |
| 変更履歴を見る（簡易） | `git log --oneline -10` |
| 変更履歴を見る（詳細） | `git log -5` |
| 特定ファイルの変更を見る | `git diff ファイル名` |
| GitHubにプッシュ | `git push` |
| GitHubから最新を取得 | `git pull` |

### トラブル時

| やりたいこと | コマンド |
|---|---|
| 認証状態の確認 | `gh auth status` |
| 再ログイン | `gh auth login` |
| ファイルを直前の状態に戻す | `git checkout -- ファイル名` |
| デプロイ一覧の確認 | `npx vercel ls` |
| リポジトリの情報を表示 | `gh repo view` |

### Claude Code 専用

| やりたいこと | 方法 |
|---|---|
| 手動コミット | `/commit` と入力 |
| 変更の確認 | 「git status を見せて」と伝える |
| 履歴の確認 | 「最近の変更履歴を見せて」と伝える |
| 変更を戻す | 「直前の変更を元に戻して」と伝える |

## 8. 用語集

| 用語 | 読み方 | 意味 |
|---|---|---|
| **Git** | ギット | ファイルの変更履歴を管理するツール |
| **GitHub** | ギットハブ | Gitの履歴をクラウド上に保存・管理するサービス |
| **リポジトリ** | - | プロジェクトの保管場所（フォルダのようなもの） |
| **commit** | コミット | 変更を記録すること（セーブのようなもの） |
| **push** | プッシュ | ローカルの変更をGitHubに送ること |
| **pull** | プル | GitHubの変更をローカルに取り込むこと |
| **clone** | クローン | GitHubからプロジェクトをコピーすること |
| **branch** | ブランチ | 変更を分岐させる機能（現在は未使用） |
| **merge** | マージ | 分岐した変更を統合すること |
| **conflict** | コンフリクト | 同じ箇所を別々に編集してしまった時の衝突 |
| **.gitignore** | ギットイグノア | Gitに含めないファイルを指定する設定ファイル |
| **Private** | プライベート | 自分だけがアクセスできるリポジトリ |
| **Public** | パブリック | 誰でも閲覧できるリポジトリ |
| **Vercel** | ヴァーセル | Webサイトをホスティング（公開）するサービス |
| **デプロイ** | - | アプリを公開サーバーに配置すること |
| **hook** | フック | 特定のタイミングで自動実行される処理 |
